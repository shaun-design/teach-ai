Insight cards

09 · Key design decisions

Show the thinking, not just the pixels

Decision snapshot
ProblemTeachers weren’t engaging with long video review.
Options(1) Video-first, (2) Timestamp jumps, (3) Insight-first summary.
DecisionInsight-first: reduce time-to-value; video supports the insight, not the reverse.
Automation vs control

More automation can erode coach trust. The product explicitly treats AI output as editable drafts and surfaces review state to teachers.

11 · Impact & outcomes

How we’d measure success

Prototype-stage work uses directional targets; production would validate with usage data and coach interviews.

Success indicators (directional)
SignalBefore (typical)Target / after
Time to first feedback passDays< 5 minutes for AI-assisted first pass
Time to “what do I change?” clarityLong video reviewMinutes scanning insight summary
Coach prep per teacher touchpointHigh manual synthesisReduced via editable drafts + triage
Hypothesis mapping

If we reduce feedback latency and increase specificity → teachers apply changes sooner → instructional quality improves faster than with batched observation cycles alone.
